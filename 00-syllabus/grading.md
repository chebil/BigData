## Lab Assignments (50%)

### Lab Scoring Rubric

Each lab is graded on the following criteria:

| Criteria | Weight | Description |
|----------|--------|-------------|
| **Correctness** | 40% | Code produces accurate results, algorithms implemented correctly |
| **Code Quality** | 30% | Well-organized, commented, follows Python best practices |
| **Analysis** | 20% | Meaningful insights, interpretations, and visualizations provided |
| **Completeness** | 10% | All sections attempted, requirements met |

### Lab List and Weights

| Lab | Title | Weight | Focus |
|-----|-------|--------|-------|
| Lab 0 | Environment Setup | Pass/Fail | Docker, Jupyter, Python |
| Lab 1 | Data Exploration | 5% | EDA, data profiling |
| Lab 2 | Python & Pandas | 5% | DataFrame operations |
| Lab 3 | Statistics & Visualization | 5% | Plots, hypothesis tests |
| Lab 4 | Clustering | 5% | K-means, hierarchical |
| Lab 5 | Association Rules | 5% | Apriori algorithm |
| Lab 6 | Regression | 5% | Linear, logistic models |
| Lab 7 | Classification | 5% | Decision trees, SVM |
| Lab 8 | Time Series | 5% | ARIMA, forecasting |
| Lab 9 | Text Analytics | 5% | NLP, sentiment analysis |
| Lab 10 | Apache Spark | 5% | PySpark, distributed ML |

### Lab Submission Guidelines

**Format**:
- Submit via GitHub repository
- Include all notebooks (.ipynb files)
- Provide any supporting code files
- Include a brief README if needed

**Naming Convention**:
- Notebooks: `lab-XX-lastname-firstname.ipynb`
- Supporting files: Clear, descriptive names

**Late Penalty**:
- 10% deduction per day late
- Maximum 3 days late accepted
- After 3 days: Zero credit
- Extensions granted for documented emergencies

### Code Quality Standards

**Expected**:
- Meaningful variable and function names
- Comments explaining complex logic
- Consistent formatting (PEP 8 style)
- Error handling where appropriate
- Efficient algorithms

**Avoid**:
- Hard-coded values (use variables/parameters)
- Redundant or dead code
- Magic numbers without explanation
- Copy-pasted code without citation

## Quizzes (20%)

### Quiz Details

- **Count**: 4 quizzes (5% each)
- **Format**: Multiple choice and short answer
- **Duration**: 60 minutes
- **Attempts**: 2 attempts allowed (higher score counts)
- **Materials**: Open book, open notes
- **Coverage**: Material from preceding weeks

### Quiz Schedule

| Quiz | Week | Coverage | Weight |
|------|------|----------|--------|
| Quiz 1 | 4 | Chapters 1-3 | 5% |
| Quiz 2 | 7 | Chapters 4-6 | 5% |
| Quiz 3 | 9 | Chapters 7-8 | 5% |
| Quiz 4 | 12 | Chapters 9-11 | 5% |

### Quiz Content

**Question Types**:
- Conceptual understanding (40%)
- Algorithm application (30%)
- Code interpretation (20%)
- Best practices (10%)

**Topics Include**:
- Theoretical foundations
- Algorithm steps and logic
- When to use specific methods
- Interpretation of results
- Common pitfalls and solutions

## Capstone Project (30%)

### Project Components

| Deliverable | Weight | Due Date | Description |
|-------------|--------|----------|-------------|
| **Proposal** | 5% | Week 12 | Problem definition, data source, methodology |
| **Progress Report** | 5% | Week 13 | Initial analysis, challenges, next steps |
| **Final Code & Analysis** | 15% | Week 14 | Complete implementation and documentation |
| **Presentation** | 5% | Week 14 | 10-15 minute presentation of findings |

### Capstone Requirements

Your project must:
1. Follow the complete Data Analytics Lifecycle
2. Use a real-world dataset (minimum 1000 observations)
3. Implement at least 3 different analytical techniques
4. Include meaningful visualizations
5. Provide actionable insights or predictions
6. Be fully reproducible with clear documentation

### Detailed Rubrics

#### Proposal (5%)
- **Problem Definition** (2%): Clear, well-motivated problem
- **Data Source** (1%): Appropriate dataset identified
- **Methodology** (2%): Sound analytical approach

#### Progress Report (5%)
- **Data Exploration** (2%): Thorough EDA completed
- **Initial Analysis** (2%): At least one technique implemented
- **Challenges** (1%): Obstacles identified with mitigation plans

#### Final Submission (15%)
- **Technical Implementation** (6%): Correct, efficient code
- **Analysis Quality** (4%): Deep insights, multiple techniques
- **Documentation** (3%): Clear README, comments, report
- **Reproducibility** (2%): Others can run your code

#### Presentation (5%)
- **Content** (2%): Covers all key aspects
- **Clarity** (1.5%): Clear communication
- **Visualizations** (1%): Effective charts and graphs
- **Time Management** (0.5%): Within 10-15 minutes

## Academic Integrity

### Collaboration Policy

**Allowed**:
- Discussing concepts and approaches
- Helping debug specific errors
- Sharing resources and tutorials
- Study groups for quiz preparation

**Not Allowed**:
- Copying code from classmates
- Sharing complete solutions
- Using solutions from previous years
- Plagiarizing from online sources

### Citation Requirements

**Must cite**:
- Code from Stack Overflow or tutorials
- Dataset sources
- External libraries not in requirements.txt
- Collaboration with specific classmates

**Citation format**:
```python
# Source: https://stackoverflow.com/...
# Author: Jane Doe
# Modified for this project
```

### Violations

**First offense**: Zero on assignment + warning
**Second offense**: Zero in course + academic misconduct report
**Severe cases**: Immediate course failure + university disciplinary action

## Regrade Requests

### Process

1. **Wait 24 hours** after grade posted (cool-off period)
2. **Review feedback** thoroughly
3. **Submit request** via email within 7 days
4. **Provide justification** with specific evidence
5. **Await response** (3-5 business days)

### Important Notes

- Entire assignment may be regraded (grade can go down)
- Decision is final after regrade
- Clerical errors corrected immediately
- Regrade for grade disputes only, not for learning purposes

---

## Grade Challenges

For formal grade appeals:
1. Follow university policy
2. Provide substantial evidence of error
3. Request within semester deadlines
4. Go through appropriate channels

## Tips for Success

### High-Quality Labs
- Start early, test incrementally
- Comment your code thoroughly
- Create meaningful visualizations
- Provide interpretation, not just output
- Double-check requirements

### Quiz Preparation
- Review lecture notes and textbook
- Practice with end-of-chapter exercises
- Understand concepts, not just memorize
- Use two attempts strategically
- Join study groups

### Capstone Excellence
- Choose a topic you're passionate about
- Start planning in Week 1
- Meet proposal deadlines
- Get feedback early and often
- Document as you go, not at the end
